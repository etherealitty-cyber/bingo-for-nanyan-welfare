# 正式数据准备

正式活动需要两个 CSV 文件。请勿把包含真实姓名和问卷答案的文件提交到 GitHub。

## 参与者名单

以 `data/participants-template.csv` 为模板：

- `nickname`：活动中公开显示的唯一昵称，不能重名。
- `role`：只能填写 `camper`、`counselor` 或 `staff`。
- `eligible_for_prize`：营员填写 `1`，其他身份填写 `0`。

系统导入时会为每个人生成内部编号和唯一6位邀请码。邀请码只在活动开始前单独发给对应参与者。

## 问卷答案

以 `data/answers-template.csv` 为模板：

- 第一列昵称必须和参与者名单完全一致。
- 后续24列对应中心免费格之外的兴趣项目。
- 每个答案只能是“是”或“否”。
- 不建议接受“未作答”，否则该用户不能被用于对应格子。

## 安全要求

- 原始问卷不能进入 `src`、`public` 或 `dist`。
- 正式数据只导入 Cloudflare D1。
- 导入完成后保留一份离线备份，不要公开分享。
- 活动结束并完成复核后删除个人明细。

## 生成导入文件

将正式CSV文件准备好后执行：

```bash
npm run data:prepare -- /路径/参与者.csv /路径/问卷答案.csv
```

系统会在被Git忽略的 `.private-data` 目录生成：

- `import.sql`：导入D1的正式名单、邀请码哈希和问卷答案。
- `invite-codes.csv`：发放给参与者的明文邀请码。

明文邀请码只会出现在本地生成的CSV中，不会进入数据库或前端代码。
